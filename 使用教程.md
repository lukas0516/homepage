## 一、先搞清楚几个名词

| 名词 | 白话解释 |
| --- | --- |
| **Hugo** | 一个「把 Markdown 文本 + 配置文件 → 生成整个网站」的工具。你只负责写文字，它负责排版。 |
| **HugoBlox Academic CV** | 我们用的学术主页模板（前身叫 Wowchemy / Academic）。它规定好了「个人介绍、经历、论文」这些板块的结构。 |
| **Markdown（`.md` 文件）** | 一种纯文本格式，比 Word 简单。`# 标题`、`**加粗**`、`[链接](url)` 就是全部语法。 |
| **YAML（`.yaml` 文件）** | 一种「配置文件」格式，长得像缩进好的冒号句子，比如 `name: 王群`。**缩进必须用空格，不能用 Tab。** |
| **Terminal（终端）** | macOS 上那个黑底或白底的命令行窗口。按 `⌘ + 空格`，输入 `Terminal` 回车就能打开。 |
| **git** | 保存文件历史的工具，可以"后悔"，回到任何一个旧版本。 |

### 文件夹长什么样

打开 `00_Homepage` 文件夹，你会看到一堆东西，**你经常碰的只有这几个**：

```
00_Homepage/
├── config/_default/
│   ├── hugo.yaml          ← 站点 URL、语言
│   ├── params.yaml        ← 站点名字、简介、主题色      ⭐常改
│   └── menus.yaml         ← 顶部导航栏
├── data/authors/
│   └── me.yaml            ← 你的个人资料（姓名、简介、教育、工作、技能） ⭐⭐最常改
├── assets/media/authors/
│   └── avatar.JPG         ← 你的头像                    ⭐
├── content/
│   ├── _index.md          ← 首页布局（"我的研究"那段文字也在这里）
│   ├── experience.md      ← 经历页（实际数据来自 data/authors/me.yaml）
│   ├── publications/      ← 论文（每篇一个子文件夹）    ⭐常改
│   ├── projects/          ← 项目（每个一个子文件夹）    ⭐常改
│   ├── blog/              ← 博客文章
│   ├── courses/           ← 课程
│   └── events/            ← 演讲、活动
└── static/uploads/
    └── resume.pdf         ← 可下载的简历 PDF
```

> 💡 **速记**：改个人信息 → 改 `data/authors/me.yaml`；加论文 → 在 `content/publications/` 下新建文件夹；改首页文字 → 改 `content/_index.md`。其他目录（`layouts/`, `go.mod`, `node_modules/`, `public/`, `resources/`）**基本不用管**。

## 二、日常三件事：预览 → 改 → 保存

每次要更新主页，都是这三步循环。

### 步骤 1：启动预览服务器

打开 Terminal，复制下面两行（**整段一起粘贴**）：

```bash
cd "/Users/wangqun/Library/Mobile Documents/com~apple~CloudDocs/00_Homepage"
pnpm dev
```

等到屏幕上出现这一行：

```
Web Server is available at http://localhost:1313/
```

就可以打开浏览器访问 **http://localhost:1313/** 查看你的主页了。

> 💡 **保持 Terminal 窗口开着。** 这个窗口关掉 = 服务器停掉 = 网页打不开。

### 步骤 2：改内容

用你喜欢的编辑器打开要改的 `.md` 或 `.yaml` 文件，改完**保存**（`⌘ + S`）。

推荐编辑器（选一个）：

- **VS Code**（免费，最常用）：[下载](https://code.visualstudio.com/)
- **macOS 自带的「文本编辑.app」**：也能用，但没有语法高亮
- **Cursor / Zed** 等等

保存后，浏览器**会自动刷新**，不用手动按 F5。

### 步骤 3：保存历史（git 提交）

改得差不多、觉得这个版本可以保留时，**新开一个 Terminal 窗口**（`⌘ + N`），运行：

```bash
cd "/Users/wangqun/Library/Mobile Documents/com~apple~CloudDocs/00_Homepage"
git add -A
git commit -m "一句话说明你改了什么"
```

比如：

```bash
git commit -m "更新个人简介和联系方式"
git commit -m "添加 2024 年发表的论文"
```

这样以后就算改坏了，也能找回每一次"存档点"。

### 停止预览服务器

不用的时候，回到运行 `pnpm dev` 的那个 Terminal 窗口，按 **`Ctrl + C`**（注意不是 `⌘ + C`）停止。

## 三、常用修改任务

### 3.1 改个人资料（名字、简介、社交账号）

文件：[`data/authors/me.yaml`](data/authors/me.yaml) —— **这是你的个人主页核心**。

打开后你会看到结构化的 YAML 字段。下面这张表是新手最常碰的几段，照着已有内容仿写就好：

```yaml
name:
  display: Qun Wang        # 显示名（英文）
  alternate: 王群           # 中文名
role: Postdoctoral Fellow  # 头衔

bio: |                     # 一段自我介绍（| 后面换行写多行）
  Qun Wang is a Postdoctoral Fellow at ...

affiliations:              # 单位（可以多个）
  - name: Hong Kong Polytechnic University
    url: https://www.polyu.edu.hk/aae/

links:                     # 联系方式 / 社交账号
  - icon: at-symbol
    url: mailto:qunwang@polyu.edu.hk
  - icon: academicons/google-scholar
    url: https://scholar.google.com/citations?user=XXX
  - icon: brands/github
    url: https://github.com/lukas0516

interests:                 # 研究兴趣（5–6 条，每条一行）
  - Autonomous Electrified Vehicles
  - Reinforcement Learning

education:                 # 教育（按时间倒序）
  - degree: PhD in Mechanical Engineering
    institution: Nanjing University of Science and Technology
    start: 2018-09-01
    end: 2024-06-30

experience:                # 工作经历
  - role: Postdoctoral Fellow
    org: Hong Kong Polytechnic University
    start: 2024-08-01
```

**改法**：

1. 找到要改的字段，把冒号右边的内容换成你的。
2. 想加一条（比如多一段经历）：复制最近一条整段，改里面的内容即可。**注意保持缩进对齐**。
3. 保存后浏览器自动刷新。

> ⚠️ **YAML 三条铁律**：
> - 缩进**只能用空格**（不能用 Tab），同层级缩进数要一致（一般 2 个空格）。
> - 冒号后**必须有一个空格**：`name: 王群` ✅，`name:王群` ❌。
> - 字符串里如果有 `:` `#` `"` 这种特殊符号，整段用双引号包起来。

### 3.2 换头像

1. 准备一张正方形头像图片（推荐 ≥ 500×500 像素）。
2. 命名为 **`avatar.jpg`**（**注意：后缀字母全小写**），放到 `assets/media/authors/` 目录下。
3. 替换现有的 `avatar.JPG` 文件（顺手把这个大写后缀的旧文件删掉）。
4. 不需要在 yaml 里写文件名，Hugo 会自动找名字叫 `avatar.*` 的图片。

> ⚠️ **小坑提示**：现在仓库里头像后缀是大写 `.JPG`。本地 macOS 不区分大小写，看起来正常；但 Cloudflare 的服务器是 Linux **区分大小写**，可能导致线上图片加载不出来。建议尽快换成全小写 `avatar.jpg`。

### 3.3 添加一篇论文

每篇论文是 `content/publications/` 下的**一个子文件夹**。

1. 在 `content/publications/` 下**复制一个已有的论文文件夹**（推荐 `adaptive-leading-cruise-2023/`），改名为你的论文短名（英文小写、用连字符）：

   ```
   content/publications/my-new-paper-2025/
     └── index.md
   ```

2. 打开 `index.md`，按下面的模板修改 frontmatter：

   ```yaml
   ---
   title: "你的论文标题"
   authors:
     - me                     # me 代表"你自己"，会自动链上你的个人页
     - 合作者姓名 1
     - 合作者姓名 2
   date: "2025-05-01"
   publication_types: ["article-journal"]   # 期刊文章用这个；会议论文用 "paper-conference"
   publication: "*IEEE Transactions on XXX*"   # 期刊名（用 * 号包起来表示斜体）
   abstract: |
     论文摘要……
   tags:
     - 关键词 1
     - 关键词 2
   featured: true             # 设为 true 后，这篇会显示在首页"Featured Publications"
   links:
     - type: pdf
       url: https://ieeexplore.ieee.org/document/XXX
     - type: preprint         # 可选，arXiv 预印本
       url: https://arxiv.org/abs/XXXX
   ---
   ```

3. 保存，浏览器里访问 http://localhost:1313/publications/ 就能看到（注意是 publication**s**，复数）。

> 💡 **小贴士**：文件夹名只能用英文小写字母 + 数字 + 连字符 `-`，因为它会变成网址的一部分（比如 `/publications/my-new-paper-2025/`）。

### 3.4 添加一段经历（学历/工作）

经历的**数据**（学校、公司、时间）存在 [`data/authors/me.yaml`](data/authors/me.yaml) 的 `education:` 和 `experience:` 段，按时间倒序排列。

**改法**：在已有条目下面复制一段，改时间、机构、职位即可。例如新增一份工作：

```yaml
experience:
  - role: Postdoctoral Fellow            # ← 已有的
    org: Hong Kong Polytechnic University
    start: 2024-08-01
  - role: 你的新职位                        # ← 复制一段、改这里
    org: 公司/学校名
    start: 2026-09-01
```

> 💡 [`content/experience.md`](content/experience.md) 是页面**布局**文件（决定经历页长什么样），里面没有真正的数据，**一般不用改**。

### 3.5 写一篇博客

在 `content/post/`（或 `content/blog/`）下新建一个文件夹，比如 `my-first-post/`，里面放 `index.md`：

```markdown
---
title: 我的第一篇博客
date: 2026-04-17
---

正文从这里开始……
```

### 3.6 改顶部导航菜单

文件：[`config/_default/menus.yaml`](config/_default/menus.yaml)

想去掉某一项，就把对应那段删了或注释掉（行首加 `#`）。想加一项，仿着已有条目写。

### 3.7 放一份简历 PDF 供下载

把你的简历 PDF **覆盖** `static/uploads/resume.pdf` 即可。访问 http://localhost:1313/uploads/resume.pdf 就能下载。

### 3.8 换主题色

文件：[`config/_default/params.yaml`](config/_default/params.yaml)，找到 `theme.colors`：

```yaml
theme:
  colors:
    primary: "indigo"     # 可填 Tailwind 颜色名（red/blue/emerald/...）或 #6366f1 这种十六进制
```

保存后浏览器自动刷新看效果。

### 3.9 让网站显示最新内容 —— 最重要的一步！⭐

改完文件、`git commit` 之后，**网上的网站还不会更新！** 因为：

- `git commit` 只是把改动存到了**你电脑里的本地仓库**。
- 真正在跑网站的服务器（Cloudflare Pages）是从 **GitHub** 上拉代码的。
- 所以你必须**把改动推到 GitHub**，Cloudflare 看到后才会重新构建发布。

**比喻**：你把作业本上的内容改了，但还没交给老师。`git push` 就是「交作业」这一步。

**完整流程**（每次改完执行一次）：

```bash
cd "/Users/wangqun/Library/Mobile Documents/com~apple~CloudDocs/00_Homepage"
git add -A
git commit -m "改动说明"
git push origin main          # ⭐ 多了这一步！把改动推到 GitHub
```

执行 `git push` 之后会发生什么：

1. GitHub 上的仓库（https://github.com/lukas0516/homepage）会更新。
2. Cloudflare Pages 检测到更新，自动开始构建（约 1–2 分钟）。
3. 构建成功后，刷新 **https://homepage-8ri.pages.dev** 就能看到新内容了。

**怎么查看部署进度？**

打开 https://dash.cloudflare.com → 左侧菜单 **Workers & Pages** → 点 `homepage` 项目 → **Deployments** 标签。能看到正在构建的版本（黄色 ●）和已发布的版本（绿色 ●）。如果是红色 ●，说明构建失败，点进去看日志或者把日志贴给我。

## 四、git 新手小抄

只需要记住 4 条命令：

| 命令 | 干什么 |
| --- | --- |
| `git status` | 看看我改了哪些文件（随时可运行，不改任何东西） |
| `git add -A` | 把**所有**改动加入"待提交清单" |
| `git commit -m "说明"` | 把待提交清单打包成一个"存档点" |
| `git log --oneline` | 看历史存档点列表 |

### 后悔药

```bash
# 1. 我把某个文件改坏了，还没 add，想恢复到上次提交的样子：
git checkout -- 文件路径

# 2. 我 add 了但还没 commit，想取消 add：
git restore --staged 文件路径

# 3. 我要查看某次旧提交都改了什么：
git show 提交哈希前7位      # 哈希从 git log 里找
```

> ⚠️ **暂时先不要用** `git reset --hard`、`git push --force` 这类命令 —— 它们**会删除历史**，新手误用容易丢东西。需要用时先问一下。

## 五、常见问题 FAQ

### Q1：`pnpm dev` 报错 "command not found: pnpm"

Terminal 里运行：

```bash
npm install -g pnpm
```

再重试。

### Q2：启动后浏览器打开 http://localhost:1313/ 显示"无法访问"

- 检查 Terminal 里是不是出现了 `Web Server is available at ...`，**没出现就是还在启动**，等几秒。
- 如果 Terminal 里有红色 `ERROR`，把报错复制给我看。

### Q3：改了文件，浏览器没反应

- 检查**跑 `pnpm dev` 的那个 Terminal 窗口是不是还开着**。关了就没预览了。
- 看那个窗口里有没有新输出 `Change detected`，没有说明文件没真正保存（按 `⌘ + S`）。
- 强制刷新浏览器：`⌘ + Shift + R`。

### Q4：`.yaml` 文件改完报错

99% 是**缩进问题**：

- YAML **只认空格**，不认 Tab。
- 同一层级的缩进要一致（通常 2 个空格）。
- 冒号后要有一个空格：`name: 王群` ✅，`name:王群` ❌。

### Q5：iCloud 会不会把我的代码同步乱？

项目在 iCloud 里本身没问题，但我们已经做了防护：
- `public/`、`resources/`、`node_modules/` 这三个会频繁变化的目录**已被 iCloud 排除同步**（命令：`brctl evict`）。
- `.gitignore` 会让 git 忽略它们。

如果换台 Mac 打开项目，只需重新跑一次：

```bash
pnpm install
hugo mod get -u
```

### Q6：我不小心删了文件怎么办？

```bash
git status              # 确认是不是真的被删了
git checkout -- 文件路径   # 从上次提交恢复
```

只要你之前 commit 过，就能找回来。

### Q7：网站已经部署在哪里？

主页已经部署在 **Cloudflare Pages**，公开网址是：

> 🌐 **https://homepage-8ri.pages.dev**

每次你 `git push origin main` 后，Cloudflare 会自动重建并发布（1–2 分钟）。详细流程见上面的 [3.9 让网站显示最新内容](#39-让网站显示最新内容--最重要的一步)。

要换成自己的域名（比如 `wangqun.com`）？告诉我，我帮你接。
